Movie enthusiasts want to be able to access information about different movies, directors, and genres. The server-side of the myFlix application will ensure they have access to this information, that their requests can be processed, and that all necessary data can be stored.

Business Logic URL HTTP Method Query Parameters Request Body Data Format Response Body Data Format
Returns a list of all movies /movies GET None None A JSON object of all the movies
Returns data about single movie by title /movies/:title GET :title None A JSON object of a single movie
Returns data about a genre by name/title /movies/genres/:genreName GET :genreName None A JSON object of a specific genre
Returns data about a director by name /movies/directors/:name GET :directorName None A JSON object of a specific director
Register new user /users POST None A JSON object of the new user A message showing if registration was created or not
Allows users to update their user info /users/:id PUT None A JSON of the user A message indicating the update was successful or not and what was updated
Allows a user to be deleted /users/:id DELETE :id None A message confirming the user has been deleted
Allows users to add a movie to their favorites /users/:id/:movieTitle POST :id, :movieTitle None A message confirming the movie has been added to the user's list
Allows users to remove a movie from their favorites /users/:id/:movieTitle DELETE :id, :movieTitle None A message confirming the movie has been removed from the user's list
